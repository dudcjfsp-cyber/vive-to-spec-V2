# 초보자 사고 구조화 레이어 설계안

## 1) 대상 사용자
- 비전공자
- 바이브 코딩에 익숙하지 않은 초보자
- "무엇을 요청해야 할지"부터 막히는 사용자

## 2) 설계 목표
- 막연한 아이디어를 개발 가능한 스펙으로 바꾼다.
- 초보자도 개발자에게 전달 가능한 요청문을 만든다.
- 누락/영향도를 사전에 점검해 시행착오를 줄인다.

## 3) L1-L5 레이어
1. L1 문제정의 인터뷰
- 핵심: `누가/언제/무엇을/왜/성공기준`
- 보조: 부족 정보 확인용 `추가 질문 3개` 자동 생성
- 출력: 문제정의 카드

2. L2 스펙 구조화
- 핵심: 역할, Must/Nice, 5단계 흐름, 입력 필드, 권한표
- 출력: 표준 스키마 JSON + 비전공자/개발자 스펙 문서

3. L3 요청문 변환
- 핵심: 원문을 `짧은/표준/상세` 요청문으로 변환
- 출력: 복붙 가능한 개발 요청문

4. L4 실행/검증
- 핵심: 완성도 점수, 누락 경고, 변경 영향도(화면/권한/테스트)
- 출력: 위험 신호 + 점검 체크포인트

5. L5 학습/실행
- 핵심: 오늘 할 일 3개
- 출력: 즉시 실행 가능한 액션 목록

## 5) 가벼운 와이어프레임(텍스트)
```text
[입력 영역]
  vibe 입력
  └─ 변환 실행

[결과 탭]
  비전공자 | 개발자 | 사고 | 레이어 | 용어

[레이어 탭]
  L1 -> L2 -> L3 -> L4 -> L5 (흐름 표시)
  [카드] L1 문제정의 인터뷰
  [카드] L2 스펙 구조화
  [카드] L3 요청문 변환
  [카드] L4 실행/검증
  [카드] L5 학습/실행
```

## 6) 화면별 요구사항
1. 인터뷰 화면
- 5칸 입력
- `추가_질문_3개`로 누락 정보를 추가 수집

2. 스펙/검증 화면
- 역할/기능/흐름/권한/입력필드 표기
- 완성도 점수 + 누락 경고 + 영향도 표시

3. 실행 화면
- 오늘 할 일 3개
- 요청문 3종 복사
